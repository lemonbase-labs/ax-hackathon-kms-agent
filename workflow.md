# 콘텐츠 제작·발행 워크플로우 자동화

## 전체 아키텍처

```text
[수집]                       [필터링·저장]              [큐레이션]              [초안 생성]
RSS 핵심 매체     ─┐                                  주제별 N개 누적시      SEO 분석 +
Google Alerts RSS ─┼──→  Claude API 스코어링  ──→   Claude로 앵글 제안  ──→  레퍼런스 통합 →
(키워드 기반)     ─┘     → Notion DB 적재          → 사람이 선택            섹션별 초안
                          (메타데이터 태깅)                                  → 사람이 검토

                                              [Serper API: 글감 결정 시 온디맨드]
```

## 기술 스택

- **언어/실행**: Python + cron (또는 GitHub Actions로 무료 스케줄링)
- **LLM**: Claude API (스코어링·요약·초안)
- **저장소**: Notion DB (메타데이터 풍부, 사람 검토 UI 좋음)
- **검색**: Serper API (월 2,500회 무료, Google SERP)
- **수집 채널**: RSS + Google Alerts RSS (모두 무료)
- **(선택) 노코드**: 1단계만 n8n으로 빼는 하이브리드도 가능

## 단계별 실행 플랜

### Phase 1: 수집 인프라 (1주차)

**할 일**

- 핵심 HR 매체 RSS 10~15개 선정·구독 (SHRM, HBR, Josh Bersin, HR Brew 등)
- Google Alerts에서 키워드별 알림 생성, 전송 위치를 RSS로 설정
  - 예: "employee retention", "people analytics", "organizational culture", 한국어 키워드 별도
- Notion DB 스키마 설계
  - 필드: 제목 / URL / 발행일 / 출처 / source_type(rss/alert) / 주제태그 / 관련성점수 / 한줄요약 / 핵심인용 / 상태(미검토/유망/사용)

**검증 기준**: 1주일간 수동으로 수집해보고 폭과 노이즈 확인

### Phase 2: 필터링·스코어링 자동화 (2주차)

**할 일**

- Python 스크립트 작성: RSS 파싱 → 본문 추출 → Claude API로 평가 → Notion 적재
- 스코어링 프롬프트 설계 (1~10점 + 사유 + 주제 태그)
- cron/GitHub Actions로 매일 또는 주 1회 실행
- 임계값(예: 7점 이상)만 통과시켜 노이즈 제거

**검증 기준**: 본인이 실제 클릭한 글과 점수 분포가 일치하는지 1주 관찰 후 가중치 조정

### Phase 3: 글감 큐레이션 (3~4주차)

**할 일**

- 주제 태그별로 N개(예: 5개) 이상 누적되면 Slack/이메일 알림
- 누적 글들을 Claude에게 입력 → 앵글 3~5개 제안
- 사람이 앵글 1개 선택 ← **품질 게이트 1**

**프롬프트 핵심**: 단순 요약이 아니라 "관통하는 인사이트", "기존 담론과의 차별점", "독자에게 새로운 관점이 될 앵글" 요구

### Phase 4: SEO 분석 + 아웃라인 (5주차)

**할 일**

- 선택된 앵글로 타겟 키워드 도출
- Serper API로 상위 10개 글 가져와 분석
  - 제목 패턴, H2/H3 구조, 평균 분량, 공통 서브토픽, 누락된 관점
- Phase 1~3에서 모은 레퍼런스 + SEO 분석 통합 → 아웃라인 생성
- 사람이 아웃라인 검토·수정 ← **품질 게이트 2**

### Phase 5: 초안 생성 (6주차~)

**할 일**

- 섹션(H2)별로 분리해서 Claude 호출 (한 번에 전체 X)
- 각 주장 옆에 `[출처 URL]` 강제 인용
- "한국 HR 맥락 1개 이상", "AI 티 나는 표현 금지어 리스트" 프롬프트에 포함
- 본인 과거 글 2~3편을 스타일 레퍼런스로 첨부
- 메타 패키지 생성: 제목 후보 5개, 메타 디스크립션, 발췌문, 이미지 프롬프트

**검증 기준**: 본인이 직접 쓴 글과 비교, 5편 초안 작성 후 프롬프트 튜닝

## 품질을 좌우하는 디테일

- **메타데이터를 풍부하게** — 1단계에서 한 줄 요약·핵심 인용·통계를 미리 뽑아두면 5단계 인용이 정확해짐
- **사람 개입 2번 보장** — 앵글 선택 + 아웃라인 검토. 자동화로 끝까지 가면 평범한 글이 나옴
- **레퍼런스 인용 강제** — 환각 방지 + 검증 용이
- **단일 SERP API 의존 금지** — Serper/SerpAPI/DataForSEO 전환 가능하게 추상화 레이어 둘 것
- **점진적 복잡도** — 처음엔 필터링 없이 다 모아보고 본인 클릭 패턴 학습 후 가중치 조정
